# Windows 高性能看图软件设计规格

日期：2026-06-22  
状态：已确认设计  
目标平台：Windows 10 22H2、Windows 11，x64

## 1. 产品目标

开发一款以速度为第一优先级的 Windows 原生看图软件。它应当快速启动、快速显示图片、快速切图，并能流畅浏览包含大量文件的目录和常见压缩包。

优先级如下：

1. 看图速度尽可能快。
2. 支持尽可能多的图片和压缩格式。
3. 界面简单、克制、美观。
4. 目录缩略图生成和滚动尽可能快，且尺寸可调。

首版只负责浏览，不提供删除、重命名、移动、收藏、裁剪、调色或格式转换功能。旋转仅改变当前视图，不写回文件。

## 2. 性能基线与目标

优化基准设备：

- 4 核 CPU
- 8 GB 内存
- 集成显卡
- SATA SSD

目标指标：

- 冷启动至窗口可见：不超过 150 ms。
- 常规 JPEG 首次可见：目标不超过 100 ms。
- 已预读图片切换：目标不超过 16 ms。
- 再次打开已缓存目录时，首屏缩略图可见：目标不超过 100 ms。
- 包含十万个项目的目录仍可流畅滚动。
- 后台扫描、预读和缩略图任务不得明显阻塞翻图。
- 内存使用必须有硬性预算，并能随系统内存压力自动收缩。

这些数值是性能目标，不是对任意文件、介质和压缩算法的绝对保证。验收测试必须记录文件类型、尺寸、存储介质、缓存状态和硬件配置。

## 3. 技术路线

采用 C++20 原生混合解码架构：

- 窗口和系统集成：Win32。
- 渲染：Direct3D 11 与 Direct2D。
- 系统格式支持：Windows Imaging Component（WIC）。
- 构建系统：CMake。
- 依赖管理：vcpkg manifest 模式，并提交版本基线以保证可复现构建。

不采用 Qt、Electron 等重量级界面框架。解码器、压缩包读取器、导航、缓存和界面通过稳定接口隔离，以便单独替换或优化。

## 4. 总体架构

### 4.1 组件

- `App/UI`：窗口生命周期、输入、快捷键、看图模式和缩略图模式。
- `Navigator`：将目录或压缩包统一呈现为有序图片序列。
- `FormatProbe`：读取少量文件头，按内容而不是仅按扩展名识别格式。
- `ImageLoader`：选择解码器，执行缩放解码、元数据读取和像素格式归一化。
- `ArchiveReader`：列出压缩包条目，并按需把条目解压到内存流。
- `RenderEngine`：纹理上传、缩放、平移、旋转、动画呈现和 ICC 色彩转换。
- `ThumbnailService`：生成缩略图并管理内存、磁盘两级缓存。
- `TaskScheduler`：执行可取消的优先级任务。
- `SettingsStore`：保存布局、快捷键和缓存设置。

### 4.2 关键数据流

```text
文件或压缩包条目
  → 读取少量头部并识别格式
  → 选择专用解码器
  → 解码为受控像素缓冲区
  → 应用方向与 ICC 转换
  → 上传 GPU 纹理
  → Direct3D/Direct2D 显示
```

UI 线程不得执行目录扫描、压缩解码、图片解码或缩略图生成。后台任务通过不可变结果或线程安全队列回传，过期结果不得覆盖当前选择。

## 5. 图片格式与解码

首版范围：

- 常见静态格式：JPEG、PNG、BMP、TIFF 等。
- 动图：GIF、APNG、WebP 动画。
- 新格式：WebP、AVIF、HEIC。
- 相机 RAW。
- PSD 合成图。

建议后端：

- JPEG：libjpeg-turbo。
- WebP：libwebp。
- AVIF：libavif。
- HEIC：libheif。
- RAW：LibRaw。
- BMP、TIFF、ICO 及适合由系统处理的格式：WIC。
- PNG/APNG、GIF 和 PSD 使用独立解码适配器。格式阶段开始时必须先完成候选库准入测试：覆盖目标文件特性、许可允许再分发、畸形输入不越界、内存受预算控制，并在固定语料上达到该组候选的最佳或接近最佳解码时间。通过结果及版本写入依赖清单后方可进入集成。

策略：

- JPEG 在目标显示尺寸较小时使用缩放解码，避免总是解码完整分辨率。
- RAW 优先读取内嵌预览以实现快速首显，后台按需替换为完整解码结果。
- PSD 只显示合成图，不解析或呈现编辑图层。
- 动图按时间轴播放；非当前图片停止动画，减少资源占用。
- EXIF 方向必须在显示和缩略图中一致生效。
- 超大图优先使用解码器的分级能力；不支持分级时使用受预算约束的降采样或瓦片策略。
- 首版支持 ICC 色彩管理，HDR 留待后续版本。

## 6. 压缩包支持

首版支持常见压缩与漫画包：

- ZIP、7Z、TAR 及 7-Zip SDK 能稳定读取的常见格式。
- RAR、分卷 RAR、CBR。
- CBZ。
- 加密压缩包。
- 分卷压缩包。

实现建议：

- 7-Zip SDK 负责 ZIP、7Z、TAR 等。
- UnRAR 负责 RAR、CBR 和分卷 RAR。

行为规则：

- 首次打开只读取条目目录，不整包解压。
- 只将当前条目和预读条目按需解压到内存。
- 不把完整原图作为临时文件散落到磁盘。
- 压缩包缩略图直接从内存流生成，只有缩略图结果进入磁盘缓存。
- 固实压缩包维持顺序解压会话，并围绕当前浏览位置复用结果，避免每张图都从头解压。
- 前后条目按预测方向预读，但不得挤占当前图片任务。
- 密码仅保存在当前进程会话，不写入磁盘。
- 条目使用自然排序，例如 `2.jpg` 排在 `10.jpg` 前。
- 非图片条目默认不进入浏览序列。

## 7. 调度与预读

任务优先级从高到低：

1. 当前正在观看或预览的图片。
2. 当前可见区域的缩略图。
3. 当前图片前后各 1–2 张。
4. 即将进入视口的缩略图。
5. 目录或压缩包中的其余缩略图。

调度要求：

- 所有任务可取消。
- 快速滚动时取消已离开预取窗口的缩略图任务。
- 用户改变选择后，旧图片任务即使完成也不得覆盖新图片。
- 交互期间降低后台并发，空闲时再提高吞吐量。
- 解码线程数按 CPU 核心数、格式特性和内存预算动态调整。
- 内存紧张时依次取消远端缩略图、相邻预读和高分辨率后台替换。

## 8. 缩略图系统

### 8.1 虚拟化

缩略图网格采用虚拟列表。无论目录有多少项目，UI 只实例化可见区域和小范围缓冲区内的单元。

### 8.2 两级缓存

- 内存缓存：保存当前可见区域及附近缩略图，使用 LRU 或等价策略淘汰。
- 磁盘缓存：保存已缩放的缩略图文件。
- SQLite：只保存缓存索引、来源指纹、尺寸档、访问时间和版本信息。

缓存键至少包含：

- 规范化来源路径。
- 文件大小与最后修改时间。
- 压缩包自身指纹。
- 压缩包内条目的稳定标识、大小和校验信息（可用时）。
- 缩略图尺寸档。
- 解码与色彩处理版本。

来源变化或缓存格式升级时自动失效。SQLite 损坏时自动重建，不影响原文件。

缩略图采用有限尺寸档。用户拖动尺寸滑块时先复用最接近档位并即时重排；停止拖动后再补生成清晰版本。RAW 和 PSD 优先使用内嵌预览。

设置中提供磁盘缓存上限和一键清理。缓存清理不得阻塞 UI。

## 9. 界面与交互

### 9.1 看图模式

- 默认只突出图片。
- 工具栏平时隐藏，鼠标移动或触及边缘时浮现。
- 工具栏尽量使用符号和图标，不显示常驻中文标签。
- 图标悬停显示本地化文字提示，并提供无障碍名称。
- 支持上一张、下一张、缩放、拖动画面、顺时针旋转视图、适应窗口、1:1 和全屏。
- 网格图标直接切换至缩略图模式，并保持当前图片选中。

### 9.2 缩略图模式

界面由图片预览区、可拖动分隔线和缩略图面板构成：

- 单击缩略图更新预览。
- 双击缩略图进入沉浸式看图。
- 预览区可关闭；关闭后缩略图面板占满可用空间。
- 预览区可通过图标恢复。
- 分隔线用于调整预览区与缩略图面板的面积。
- 缩略图面板可停靠在上、下、左、右四侧，默认在下方。
- 停靠方向可以通过布局图标切换，也允许拖动停靠。
- 软件记忆停靠方向、各方向对应的面板尺寸、缩略图尺寸和预览开关。
- `Ctrl + 鼠标滚轮` 和尺寸滑块均可调整缩略图大小。
- 方向键移动选择，Enter 查看，Esc 返回。

快捷键和鼠标操作允许用户自定义，但首版应提供不冲突、无需配置即可使用的默认方案。

## 10. 渲染与色彩

- GPU 负责最终缩放、平移和旋转，避免 CPU 重复处理整张图。
- 保留高质量与快速缩放路径；交互缩放时优先响应，停止后可提升采样质量。
- 首版读取嵌入式 ICC 配置，并转换至当前显示器色彩空间。
- 多显示器移动时重新选择目标显示配置。
- GPU 设备丢失时重建设备和纹理；重建失败则回退至 WARP 软件渲染。
- HDR 显示和 HDR 色调映射不属于首版范围。

## 11. 错误处理与健壮性

- 损坏或不支持的图片显示轻量占位提示，仍可继续翻图。
- 密码错误允许重试，不冻结其他浏览任务。
- 分卷缺失时指出缺失卷，不尝试无限重试。
- 网络盘或移动介质断开后保留当前已解码画面，并允许切换来源或等待恢复。
- 内存不足时取消低优先级任务、清理缓存并降低非当前内容的解码分辨率。
- 磁盘缓存和设置写入采用原子替换或事务。
- 对不可信输入执行尺寸、像素数、帧数、分配量和递归深度限制，防止解压炸弹与畸形图片耗尽资源。
- 对高风险解码器预留进程隔离接口；是否在首版启用由实施阶段的模糊测试结果决定。

## 12. 设置持久化

保存以下用户状态：

- 窗口位置、尺寸和全屏状态。
- 上次使用的模式。
- 缩略图停靠方向，默认下方。
- 每个停靠方向的面板尺寸。
- 预览区开关与比例。
- 缩略图尺寸。
- 快捷键和鼠标映射。
- 磁盘缓存上限。

不得持久化压缩包密码。配置损坏时恢复默认值，不阻止程序启动。

## 13. 测试与验收

### 13.1 单元测试

- 自然排序。
- 格式探测。
- 缓存键和失效规则。
- 任务优先级、取消和过期结果抑制。
- 目录与压缩包路径规范化。
- 布局状态保存和恢复。

### 13.2 兼容性测试

- 每种支持的静态和动画图片格式。
- EXIF 方向、ICC、透明通道和多帧文件。
- RAW、PSD、加密包、分卷包、固实压缩包。
- Unicode、超长路径和包含特殊字符的文件名。
- 本地盘、网络盘和可移动介质。

### 13.3 压力与安全测试

- 十万项目录。
- 超大尺寸图片。
- 数 GB 压缩包。
- 快速连续翻图、滚动和调整缩略图大小。
- 损坏图片、损坏缓存、缺失分卷和错误密码。
- 模糊测试解码入口与压缩包条目处理。
- 解压炸弹、异常像素尺寸和超多动画帧。

### 13.4 性能回归

建立固定的图片与压缩包语料库，记录：

- 冷启动时间。
- 首图可见时间。
- 预读命中与未命中的切图时间。
- 缩略图首屏时间和完整目录吞吐量。
- CPU、内存、GPU 和磁盘峰值。
- 缓存命中率。

关键路径出现可重复的明显回退时，持续集成测试应报警。

## 14. 首版明确不做

- 图片编辑或写回。
- 文件删除、移动、复制、重命名和收藏。
- 图层级 PSD 浏览。
- HDR 输出。
- Windows 7/8 或 32 位支持。
- 云图库、同步、标签数据库和人脸识别。

这些边界用于保护启动速度、运行时复杂度和首版交付周期。
